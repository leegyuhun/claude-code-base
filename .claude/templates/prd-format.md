# PRD 문서 템플릿

> `/prd` 커맨드 Phase 3에서 이 파일을 읽어 PRD.md를 생성한다.
> 섹션 순서와 변수 자리표시자(`{...}`)를 유지하고, 실제 값으로 치환하여 저장한다.

---

## 작성 원칙

- 각 섹션은 구체적이고 검증 가능하게 작성
- "완료 기준 (MVP)"는 Validator의 검증 계약과 연결될 수 있도록 측정 가능하게
- 우선순위(P0/P1/P2)는 orchestrator의 plan.md 기능 목록과 직결되므로 반드시 분류
- 사용자가 "모른다"고 한 항목은 "[확인 필요]"로 표기
- `ADDITIONAL_CONTEXT`가 있으면 "목적 & 배경"에 이슈 설명과 자연스럽게 통합 서술하고,
  "관련 레퍼런스" 섹션에 `**사용자 보완 메모:**` 항목으로 원문을 부속 기록(추적성 확보)
- "재현방법" 섹션은 **버그/이슈 기반 PRD에서만** 작성. 신규 기능 PRD면 섹션 전체를 생략한다.
  - Redmine 이슈가 있거나 `EXTRA_CONTEXT`에 버그/오류/재현 관련 키워드가 있으면 자동 포함
  - 재현 단계는 번호 매긴 순서대로, "기대 결과"와 "실제 결과"를 분리 기술

---
