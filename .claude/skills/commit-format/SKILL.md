# 커밋 메시지 형식 스킬

## YSR 커밋 메시지 공식 형식

```
fix #[이슈번호] 제목 (한 줄)

　# 변경 내용
　　- 변경사항 1
　　　: 부연 설명
　　- 변경사항 2
　# 관련파일
　　- AssemblyInfo.cs(버전업)
　　- 수정파일.cs(수정 내용 요약)
```

### 핵심 규칙

| 항목 | 규칙 |
|------|------|
| 첫 줄 | `fix #이슈번호 [카테고리] 제목` |
| 본문 구분 | ... 블록 |
| 들여쓰기 | 전각 공백(`　`) 사용 — 일반 스페이스 아님 |
| 계층 | `#` 대분류 → `-` 항목 → `:` 설명 |
| 언어 | 한국어 (함수명·파일명은 원어) |
| 관련파일 | `AssemblyInfo.cs(버전업)` 포함 (C# 수정 시 필수) |

카테고리가 불명확하면 버그 현상의 핵심 도메인어를 `[]`로 감싸 사용.

## 브랜치 네이밍

```
[연도]_정기5차_#[이슈번호]
예: 2026_정기5차_#207500
```

Merge 커밋 (자동 생성되지만 참고용):
```
Merge branch '2026_정기5차_#207500' into '2026_정기5차'
```

## 실제 커밋 예시 (참고)

```
fix #207231 [다빈도프린터] 김화내과 출력 지연현상

　# 변경내용
　　- Spooler 재시작 로직 주석처리
　　　: Spooler가 정상적인 상황임에도, 비정상으로 인식하여 재시작하는 케이스가 확인됨
　　　: 명확한 사유가 확인되지 않아, 우선 주석처리
　# 관련파일
　　- PrinterUtil.cs(Spooler 재시작 로직 주석처리)
```

## 작성 절차

1. 이슈 번호 파악 → Redmine 조회 또는 GOAL.md / 사용자 요청에서 확인, 카테고리와 제목 결정
2. `git diff`로 변경 내용 파악 `# 변경내용 블록 작성
3. 수정 파일 목록 → `# 관련파일` 섹션 작성
4. 전각 공백 들여쓰기 확인
5. `sprints/{sprint}/COMMIT_MESSAGE.md`에 최종 결과 저장 (스프린트 워크플로우 시)
