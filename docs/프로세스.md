1. 새 프로젝트 / 스프린트 개발

  # 처음 시작할 때
  .claude/agents/orchestrator.md와 STATUS.md를 읽고 현재 PHASE부터 실행해줘.

  # 스프린트 계획
  .claude/agents/planner.md와 STATUS.md를 읽고 현재 스프린트 GOAL.md를 작성해줘.

  # 스프린트 구현
  /sprint-dev sprint-01

  # 검증
  .claude/agents/validator.md와 STATUS.md를 읽고 현재 스프린트 검증을 시작해줘.

2. Redmine 이슈 기반 작업

  # 이슈번호로 PRD 생성 → 브랜치 생성 → 개발 착수
  /prd #207500

  # 작업 완료 후 Resolved 처리
  /resolve 207500

3. 핫픽스 (긴급 수정)

  # 1. 브랜치 생성 (직접)
  git checkout -b 2026_정기5차_hotfix_버그설명

  # 2. .pas 파일 수정 후 push + MR
  .claude/agents/deploy-prod.md와 docs/STATUS.md를 읽고 현재 hotfix 브랜치의 push + GitLab MR 안내를 진행해줘.

---
새 프로젝트 시작

Step 1. PRD.md 작성 (직접)

PRD.md에 프로젝트 요구사항을 작성해줘:
- 목적 & 배경
- 핵심 기능 및 요구사항 (폼/유닛/컴포넌트 단위)
- MVP 완료 기준
- 제약 조건 (Delphi 2007 호환성, Win32 전용 등)

Step 2. Orchestrator 실행 → plan.md + ROADMAP.md + 초기화

.claude/agents/orchestrator.md와 STATUS.md를 읽고 현재 PHASE부터 실행해줘.
[PAUSE] 지점에서 멈추고 내 확인을 기다려.
코드 구현은 하지 마. 계획 문서 작성만 해.

▎ plan.md 확인 → ROADMAP.md 확인 → CLAUDE.md 생성 (msbuild 빌드 명령 포함)

---
스프린트 반복 (메인 루프)

Step 3. Planner 실행 → GOAL.md 생성

.claude/agents/planner.md와 STATUS.md를 읽고 현재 스프린트 GOAL.md를 작성해줘.
코드 구현은 하지 마. GOAL.md 작성만 해.
완료되면 다음 에이전트 실행 방법을 알려줘.

Step 4a. Implementer 실행 (직접 구현)

.claude/agents/implementer.md와 STATUS.md를 읽고
sprints/{현재스프린트}/GOAL.md 기준으로 구현 시작해줘.
[PAUSE] 지점에서 멈추고 내 확인을 기다려.
GOAL.md 범위 밖의 기능은 구현하지 마.

Step 4b. 또는 sprint-dev 커맨드

/sprint-dev sprint-01

▎ 구현 Plan 먼저 보여준 뒤 '실행' 확인 받고 기능별 순차 구현

Step 5. Validator 실행 → msbuild 빌드검증 + DUnit + PR 생성

.claude/agents/validator.md와 STATUS.md를 읽고
sprints/{현재스프린트} 검증을 시작해줘.
[PAUSE] 지점에서 멈추고 내 확인을 기다려.

▎ build.bat 실행 → DUnit 실행 → 수동테스트 → PR 자동 생성 → 다음 스프린트

핫픽스 (긴급 수정)

# 1. 브랜치 생성 (직접)
git checkout -b {현재브랜치}_hotfix_버그설명

# 2. .pas 파일 수정 후 push + MR
.claude/agents/deploy-prod.md와 docs/STATUS.md를 읽고
현재 hotfix 브랜치의 push + GitLab MR 안내를 진행해줘.

---
상태 확인 스킬 (언제든지)

/status        ← 현재 PHASE, 스프린트, 에이전트 상태 한눈에
/next          ← 지금 뭘 실행해야 하는지 명령어 자동 생성
/sprint-log    ← 현재 스프린트 GOAL/DONE/OUT_OF_SCOPE 요약
/rollback      ← 이전 PHASE로 되돌리기
/debt          ← TODO 주석 + 기술 부채 종합 보고
/branch        ← 스프린트/핫픽스 브랜치 생성
/prd           ← PRD.md 작성 가이드
/resolve {번호} ← Redmine 이슈 Resolved 처리

---
전체 흐름 한눈에

PRD.md 작성
   ↓
Orchestrator  →  plan.md 확인  →  ROADMAP.md 확인  →  CLAUDE.md 생성
   ↓
Planner  →  GOAL.md 확인 (구현할 폼/유닛 목록)
   ↓
Implementer (또는 /sprint-dev)
→ .pas + .dfm 폼/유닛 작성
→ 비즈니스 로직 작성
→ DUnit 테스트 작성
   ↓
Validator  →  build.bat 실행  →  DUnit 실행  →  수동테스트  →  PR생성
   ↓
다음 스프린트 반복

▎ 막히면: /next 입력하면 현재 상태 기반으로 실행 명령어를 바로 알려줍니다.

### 기존 PRD.md 작업이 종료되고 새로운 요구사항이 생긴다면
  프로젝트레포/                                                                                                                 
  ├── .claude/      ← 한 번 세팅, 계속 재사용
  │   ├── agents/       ← 에이전트 역할 정의 (불변)
  │   ├── rules/        ← 코딩 원칙 (불변)
  │   └── commands/     ← 커맨드 (불변)
  │
  ├── PRD.md            ← 새 요구사항 생기면 업데이트
  ├── plan.md           ← Orchestrator가 갱신
  ├── STATUS.md         ← 항상 현재 상태 반영
  └── sprints/
      ├── ROADMAP.md    ← Orchestrator가 스프린트 추가
      ├── sprint-01/    ← 완료된 스프린트 (아카이브)
      ├── sprint-02/    ← 완료된 스프린트 (아카이브)
      └── sprint-03/    ← 진행 중

  ┌────────────┬─────────────────────────────────────┐
  │    파일    │                할 일                │
  ├────────────┼─────────────────────────────────────┤
  │ PRD.md     │ 새 요구사항 섹션 추가               │
  ├────────────┼─────────────────────────────────────┤
  │ ROADMAP.md │ Orchestrator가 새 스프린트 행 추가  │
