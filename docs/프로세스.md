---
전체 흐름 한눈에

  /prd #이슈번호
      ↓ PRD.md 생성 + 브랜치 생성 + Redmine InProgress
  Orchestrator  →  plan.md  →  ROADMAP.md  →  CLAUDE.md
      ↓
  Planner  →  GOAL.md
      ↓
  /sprint-dev  →  .pas/.dfm 구현
      ↓
  Validator  →  빌드/테스트  →  수동테스트  →  commit + push + GitLab MR
      ↓
  /resolve #이슈번호  →  Redmine Resolved

  ┌───────┬───────────────────┬────────────────────────────────┬──────────────────────────────────────┐
  │ PHASE │ 에이전트/커맨드   │ 프롬프트                       │ 산출물                               │
  ├───────┼───────────────────┼────────────────────────────────┼──────────────────────────────────────┤
  │ PRD   │ /prd              │ /prd #207500                   │ PRD_#207500.md, 브랜치, Redmine InProgress │
  │ 1~4.5 │ Orchestrator      │ (아래 참조)                    │ plan.md, ROADMAP.md, CLAUDE.md       │
  │ 5     │ Planner           │ (아래 참조)                    │ sprints/sprint-01/GOAL.md            │
  │ 6     │ /sprint-dev       │ /sprint-dev                    │ 코드 구현, 스프린트 브랜치           │
  │ 7~10  │ Validator         │ (아래 참조)                    │ DONE.md, commit, push, GitLab MR     │
  │ 완료  │ /resolve          │ /resolve 207500                │ Redmine Resolved 100%                │
  └───────┴───────────────────┴────────────────────────────────┴──────────────────────────────────────┘

  막히면: /next 입력 → 현재 PHASE 기반 실행 명령어 자동 안내

---
단계별 상세 (예시: Redmine #207500, 현재 브랜치 main_delphi)

Step 1. PRD 생성  (/prd)

  /prd #207500

  → Redmine 이슈 자동 조회 (제목/설명/카테고리/우선순위)
  → 기술 분석 (변경 예상 파일, 공유 유닛 여부, DB 변경 여부)
  → 부족한 항목만 GAP 질문
  → docs/PRD_#207500.md 생성
  → [PAUSE] 브랜치 생성 여부 선택
       [1] main_delphi_#207500 브랜치 생성 (권장)
       [2] 현재 브랜치에서 작업
  → Redmine 상태 → InProgress
  → [PAUSE] Orchestrator 바로 시작할까요?

  산출물: docs/PRD_#207500.md, 브랜치 main_delphi_#207500

---
Step 2. 계획 수립  (Orchestrator · PHASE 1~4.5)

  /prd 완료 후 Orchestrator 자동 시작 (또는 수동 실행):

  .claude/agents/orchestrator.md와 docs/STATUS.md를 읽고 현재 PHASE부터 실행해줘.
  [PAUSE] 지점에서 멈추고 내 확인을 기다려. 코드 구현은 하지 마.

  PHASE 1 — PRD 검증
    → 필수 섹션 확인 (목적/기능/완료기준/제약조건)
    → 통과 시 PHASE 2 진행

  PHASE 2 — plan.md 생성
    → PRD 분석 → 기능 목록 (우선순위 P0/P1/P2, 복잡도 S/M/L)
    → plan.md 저장

  PHASE 3 — plan.md 검토 [PAUSE]
    → plan.md 내용 요약 출력
    → 수정 있으면 직접 편집 후 '계속' 입력

  PHASE 4 — ROADMAP.md 생성
    → 스프린트 분할 (스프린트당 3~5일, P0 먼저)
    → sprints/ROADMAP.md 저장

  PHASE 4.5 — 프로젝트 초기화 + CLAUDE.md [PAUSE]
    → 프로젝트 이미 초기화됐나요? → '예'
    → CLAUDE.md 생성 방법 선택 (claude /init 권장)

  산출물: plan.md, sprints/ROADMAP.md, CLAUDE.md

---
Step 3. 스프린트 계획  (Planner · PHASE 5)

  .claude/agents/planner.md와 docs/STATUS.md를 읽고 현재 스프린트 GOAL.md를 작성해줘.
  코드 구현은 하지 마. GOAL.md 작성만 해.

  → ROADMAP.md 기반으로 현재 스프린트 분석
  → 이슈 상세 재조회 (필요 시)
  → sprints/sprint-01/GOAL.md 생성
       - 구현 기능 체크리스트 (예상 소요 시간 포함)
       - 검증 계약 (자동/수동 항목)
       - 예상 산출물 (생성/수정 파일 목록)
       - 수동 테스트 시나리오

  산출물: sprints/sprint-01/GOAL.md

---
Step 4. 구현  (/sprint-dev · PHASE 6)

  /sprint-dev

  → 브랜치 자동 생성: main_delphi_#207500_sprint-01
  → 구현 Plan 출력 (순서 + 접근 방식) [PAUSE]
  → '실행' 입력 → GOAL.md 체크리스트 순서대로 기능 구현
       ✅ 기능 1 구현 완료
       ✅ 기능 2 구현 완료
       ...
  → 최종 빌드 확인
  → PHASE 7로 전환

  브랜치: main_delphi_#207500_sprint-01
  커밋 없음 — Validator가 완료 시점에 한 번만 커밋

---
Step 5. 검증 + MR 생성  (Validator · PHASE 7~10)

  .claude/agents/validator.md와 docs/STATUS.md를 읽고
  sprints/sprint-01 검증을 시작해줘.
  [PAUSE] 지점에서 멈추고 내 확인을 기다려.

  PHASE 7 — 자동 검증
    → build.bat debug 실행 (실패 시 최대 3회 자동 수정)
    → run_tests.bat 실행 (DUnit)
    → GOAL.md 검증 계약 독립 채점 ([ ] → [x])
    → 코드 리뷰 스코어카드

  PHASE 8 — 수동 테스트 [PAUSE]
    → GOAL.md 수동 테스트 시나리오 출력
    → EXE 실행 후 직접 테스트
    → '통과' 또는 '수정 필요: {내용}' 입력

  PHASE 9 — 커밋 + push + GitLab MR 생성
    → DONE.md 생성 (완료 기능, 변경 파일, 기술 부채)
    → CHANGELOG.md 업데이트
    → TECH_DEBT.md 업데이트
    → commit (commit-writer가 한국어 메시지 자동 작성)
    → git push -u origin main_delphi_#207500_sprint-01
    → GitLab MR 자동 생성 (타겟: main_delphi_#207500)
         glab mr create (설치 시) → GitLab API → 수동 안내 폴백
    → [PAUSE] MR 확인 후 '머지 완료' 입력

  PHASE 10 — 다음 스프린트 또는 완료
    → 스프린트가 더 있으면: CURRENT_SPRINT 업데이트 → PHASE 5로 (Step 3 반복)
    → 모든 스프린트 완료 시: "MVP 달성!" 안내

  산출물: DONE.md, CHANGELOG.md, commit, push, GitLab MR

---
Step 6. Redmine 완료 처리  (/resolve)

  /resolve 207500

  → 작업 기간 자동 계산 (첫 커밋일 ~ 오늘)
  → [PAUSE] 확인 + 코멘트 입력 (선택)
  → Redmine 상태 → Resolved, done_ratio=100%

브랜치 전략

  이슈번호 있을 때:

    main_delphi                               ← 현재 브랜치
      └── main_delphi_#207500                 ← /prd 생성 (이슈 베이스)
            ├── main_delphi_#207500_sprint-01 ← /sprint-dev 생성 → MR → main_delphi_#207500
            └── main_delphi_#207500_sprint-02 ← (다음 스프린트)   → MR → main_delphi_#207500

  이슈번호 없을 때:

    main_delphi
      └── main_delphi_sprint-01               ← /sprint-dev 생성

---
유틸리티 커맨드

  /next          ← 지금 뭘 실행해야 하는지 명령어 자동 안내
  /status        ← 현재 PHASE, 스프린트, 에이전트 상태 한눈에
  /rollback      ← 이전 PHASE로 되돌리기
  /sprint-log    ← 현재 스프린트 GOAL/DONE/OUT_OF_SCOPE 요약
  /debt          ← TODO 주석 + 기술 부채 종합 보고
  /branch        ← 브랜치 수동 생성
